The Problem

You can add properties and methods to an object. Then create new objects. There is code some reuse. However when you have multiple objects, every object has it's own copy of properties (ok) and functions (not ok). This is very inefficient as it takes up uncessary space and could ultimately affect memory and performance.

Unlike C# / C++, javascript doesnt use a class heirarchy. javascript uses an "object model" based on prototypes (equiavalent of a base class).

The Solution

Every object has a prototype. Properties and functions that belong to all instances of an object are added to this prototype.

When a property or function is referenced, javascript first looks at the current object. If it can't find it, javascript looks at the prototype. Because of this behavior, properties and methods defined in the derived object can "override" the properties and methods in the prototype (base). Unlike C++/C# (where base and derived classes are created), override occurs AFTER the object is created.
