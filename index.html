Code Reuse != Efficiency

Code reuse is essential for developing code quickly and reducing maintenance. However, reuse doesnt guarantee efficiency. For example, you can add properties and methods to a "template" object. Then derive new objects from this "template". There is code reuse. However when you have multiple objects, every object has it's own copy of properties (generally ok) and functions (not ok). This is very inefficient as it takes up uncessary space and could ultimately affect memory and performance.

Unlike C# / C++, javascript doesnt use a class inheritance. javascript uses an "object model" based on prototypes (equivalent of a base class).

Prototype - the "base class" in javascript

In javascript, every object has a prototype. Properties and functions that are shared with all instances of an object are added to this prototype.

When a property or function is referenced, javascript first looks at the current object. If it can't find it, javascript looks at the prototype. Because of this behavior, properties and methods defined in the derived object can "override" the properties and methods in the prototype (base). Unlike C++/C# (where base and derived classes are created), override occurs AFTER the object is created.

Overriding properties

Overriding properties creates a "shadow" that hides the property in the prototype. In this example when fly() is called, the object (this) is dynamically augmented with a new instance property. When the instance property is referenced, javascript looks first at the object that called fly(). Since it finds the property in the object, the property in the prototype is ignored. You can still access the base property through the prototype.

Setting default values efficiently

Suppose you want to set a flag to a default value. You could just set default values in the constructor. Although this works, every object has it's own property. And, initially they all have the same value. This is inefficient use of space. A better solution can be done by setting defaults in the prototype (then all objects share same property).
